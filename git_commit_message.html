git commit command

git commit command is really useful for take note about git commit.
we can use git commit to write message or
we can use git commit -m "the text you want as message" command to write message.
we can edit git commit template by editing the file at ~/.gitmessage.txt

If we want git commit command to use default in Visual Studio Code,
"git config --global core.editor "code --wait" this command will be make sure that default editor to VScode.

git commit -m command is really strict to commit a message. A single line is ok to use this command.
Using git commit is really useful when you want to commit a full message.

Useful command of git commit

log, shortlog commands are use to show git messages.
git log will show you full message of you commit.
git shortlog will only show you the first line of you commit.
git blame, git revert, git rebase commands will show you the list of options for git command.
git log --oneline will show you only subject line of the message.
Its similar to git shortlog


The seven rules of git commit message!

  1. Separate subject from body with a blank line
  2. Limit the subject line to 50 characters
  3. Capitalize the subject line
  4. Do not end the subject line with a period
  5. Use the imperative mood in the subject line.
  6. Warp the body at 72 characters
  7. Use the body to explain what and why vs .how is this happen!

Separate subject from body with a blank

The first line of commit message is subject and the rest is called body.

  • Firstly, not every commit requires both a subject and a body.
    Sometimes a single line is fine, especially when the change
    is so simple that no further context is necessary.
